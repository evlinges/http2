## 2. Опис використаного API

### 2.1 Компанія-розробник

Spotify Web API розроблений компанією **Spotify AB** — міжнародною компанією, що надає сервіс потокового відтворення музики, подкастів та відеоконтенту.

---

### 2.2 Опис функціональності API

Spotify Web API надає можливість взаємодіяти з платформою Spotify через HTTP-запити. За допомогою API можна:

* отримувати інформацію про артистів, альбоми та треки;
* виконувати пошук музичного контенту;
* аналізувати популярність треків;
* працювати з плейлистами та бібліотекою користувача (за наявності відповідних прав доступу).

У даній лабораторній роботі API використовується для:

* отримання access token через OAuth 2.0 (Client Credentials Flow);
* отримання інформації про артиста;
* пошуку музичних треків;
* збереження результатів пошуку у CSV-файл.

---

### 2.3 Підтримувані методи

У межах лабораторної роботи використовувалися наступні HTTP-методи:

* **POST** — для отримання токена доступу:

  * `https://accounts.spotify.com/api/token`

* **GET** — для отримання інформації:

  * `https://api.spotify.com/v1/artists/{id}` — інформація про артиста;
  * `https://api.spotify.com/v1/search` — пошук треків за ключовим словом.

Під час виконання лабораторної роботи було отримано практичні навички роботи з REST API та протоколом HTTP. Реалізовано автентифікацію через OAuth 2.0, виконано GET- та POST-запити до Spotify Web API.

У результаті було:

* успішно отримано токен доступу;
* отримано інформацію про обраного артиста;

Отримані знання дозволяють використовувати сторонні API у власних програмних проєктах та автоматизувати роботу з зовнішніми сервісами.
